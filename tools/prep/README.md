# Заготовки состояния для скриншотов

Файл делится строкой `// ПОСЛЕ:` на две части:

- до неё — выполняется, затем страница ПЕРЕЗАГРУЖАЕТСЯ (сюда ставят
  `localStorage`: тему, черновик экзамена);
- после неё — выполняется в уже запущенном приложении (сюда кладут клики,
  чтобы дойти до экрана, у которого нет адреса).

Части можно комбинировать в одном файле: сначала тема, потом клики.

Экраны, у которых АДРЕС ЕСТЬ (prep не нужен):
`#student=<uuid>` — кабинет, `#r=<uuid>` — сохранённый результат.

Экраны, требующие prep: шаги анкеты, все шесть типов вопросов,
экран «спасибо», результат сразу после отправки, вход в профиль
(`document.querySelector('#tabbar .tab:nth-child(4)').click()`).

Для экранов вопросов удобнее сначала пройти анкету (см. `wizard-last.js`),
затем нажать финальную кнопку и добавлять `document.getElementById('nextBtn')`
-подобные клики — точные id смотреть в `assets/app.js` рядом с `showStep`.
