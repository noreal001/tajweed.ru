# Заготовки состояния для скриншотов

Передаются шестым аргументом `tools/shot.mjs`. Файл делится строкой `// ПОСЛЕ:`
на две части:

- **до неё** — выполняется, затем страница ПЕРЕЗАГРУЖАЕТСЯ. Сюда кладут
  `localStorage`: тему и состояние экзамена;
- **после неё** — выполняется в уже запущенном приложении. Сюда кладут клики,
  чтобы дойти до экрана, у которого нет ни адреса, ни состояния в хранилище.

`shot.mjs` перед каждым прогоном удаляет все ключи `tajweed_*`, поэтому
состояние прошлого снимка не протекает в следующий — тему нужно задавать
в каждом «тёмном» prep-файле явно.

## Готовые файлы

| Файл | Экран |
|---|---|
| `dark.js` | главная в тёмной теме |
| `wizard-first.js` | первый шаг анкеты (имя) |
| `wizard-last.js` | последний шаг анкеты (телефон + плашка о данных) |
| `question-arabic.js` / `question-arabic-dark.js` | вопрос с арабским: сопоставление букв |
| `sifat.js` | вопрос с галочками (сифаты) |
| `compose.js` | составление слова из плиток |
| `reading.js` / `reading-dark.js` | чтение вслух и запись голоса |

## Как сделать свой

Экран экзамена задаётся индексом шага в ключе `tajweed_exam_v1` (образец —
`_state.js`). Всего 58 шагов; начала заданий:

| Шаг | Задание |
|---|---|
| 1 | сопоставление букв (арабский) |
| 3 | счёт слогов |
| 24 | сифаты (галочки) |
| 32 | составление слова |
| 40 | да/нет |
| 57 | чтение вслух и запись |

Тёмная тема добавляется строкой `localStorage.setItem('tajweed_theme','dark');`
в ту же (первую) часть файла — комбинировать с состоянием экзамена можно
свободно, обе строки идут до перезагрузки.

## Экраны с собственным адресом (prep не нужен)

- `#student=<uuid>` — личный кабинет;
- `#r=<uuid>` — сохранённый результат.

Оба uuid появляются только после реальной отправки экзамена на сервер, поэтому
для их съёмки нужен поднятый бэкенд (`ADMIN_PASSWORD=test node server.js`
в `tajweed-backend/`) и один пройденный экзамен — см. п. 3 в `PLAN.md`.

## Экраны, добываемые кликами (часть после `// ПОСЛЕ:`)

Реальные id кнопок в `assets/app.js`: `heroExam` (кнопка баннера),
`wInput`/`wNext` (поле и «Далее» в анкете), `startTask` («Начать задание»),
`answerBtn` («Сохранить ответ и продолжить»), `recBtn` (запись голоса).
Вход в профиль: `document.querySelector('#tabbar .tab:nth-child(4)').click()`.
